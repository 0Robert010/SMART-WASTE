# Documentacao SmartWaste

Esta pasta centraliza a documentacao tecnica e academica do SmartWaste.

## Documentos principais

- `projeto-upx.md`: texto base do relatorio academico da UPx.
- `checklist-upx.md`: pendencias, entregaveis e proximas tarefas.
- `arquitetura.md`: visao geral da solucao, fluxo de dados e organizacao do repositorio.
- `prototipo-fisico.md`: montagem fisica, impressao 3D, Bambu Lab A1, Bambu Studio e componentes.
- `software.md`: painel web, firmware ESP32 e plano de integracao.
- `validacao.md`: plano de testes, entrevistas e evidencias para apresentacao.
- `evidencias/`: fotos, prints e registros usados nos anexos do relatorio.

## Organizacao recomendada

- Documentos do projeto ficam em `docs/`.
- Firmware do ESP32 fica em `arduino/smartwaste_esp32/`.
- Modelos 3D e arquivos de fabricacao ficam em `modelagem-3d/`.
- Imagens e assets visuais usados no site ficam em `assets/`.
- Evidencias de desenvolvimento e testes ficam em `docs/evidencias/`.
- Codigo do painel web fica em `src/` e `index.html`.

## Padrao de escrita

- Registrar decisoes tecnicas com data quando forem definitivas.
- Evitar deixar credenciais, senhas ou dados pessoais nos documentos.
- Incluir fotos, prints ou links de evidencias quando houver teste real.
- Manter nomes de arquivos descritivos e em letras minusculas quando possivel.
